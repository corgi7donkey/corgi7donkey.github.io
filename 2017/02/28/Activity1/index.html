Activity stop状态被回收解决办法

发表于 2017-02-28 | 分类于 android

Activity1进入Activity2后Activity1进入到stop状态，这是Activity1有可能被系统回收掉。当回收掉是，用户按下back键后回到Activity1，此时并不是执行onRestart（）方法而是执行onCreate（）方法。Activity1被重新创建。

倘若Activity1存在临时数据和状态，重新创建后数据丢失，该如何是好呢？

解决办法
（1）在Activity1中重写onSaveInstanceState方法。将需要的数据存储到Bundle中。

（2）通过onCreate方法，获取bundle存储的数据
