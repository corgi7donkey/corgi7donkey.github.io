Listview优化

优化原因1：listadapter中的getView方法每次都会重新加载布局，当快速滑动时就会影响手机性能。

解决：在listadapter中getView方法里会看到convertView参数，这个参数可以将加载好的布局缓存起来。一下是代码：

优化原因2：虽然不会重新加载布局了，但是在找控件的时候，还是会调用findViewById。
解决：通过自己写的MyView来做优化。

新建了一个myView类用于缓存控件，当convertView为空时创建一个myView对象，并对myView中的属性赋值，利用setTag存放到当前view中。当convertView不为空时，通过view.getTag()获取myView对象。从而实现了控件的缓存。
